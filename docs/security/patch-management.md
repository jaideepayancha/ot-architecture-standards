# Patch and Vulnerability Management

## Objectives

- Reduce exploit exposure
- Avoid unplanned downtime

## Asset Classes

| Asset | Patch Cycle |
|--------|-----------|
| Servers | Quarterly |
| HMIs | Semi-annual |
| PLCs | As approved |

## Testing

All patches must be tested in:

DEV or TEST environment

## Vulnerability Scanning

- Passive scanning in OT only
- No active scans on PLC networks

## Emergency Patching

Allowed only with:

OT + Operations approval
